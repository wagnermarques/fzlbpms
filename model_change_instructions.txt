You have several ways to change Gemini models when using the Gemini CLI:

1.  **Using a Command-line Flag:**
    You can specify the model directly when running a command using the `--model` or `-m` flag:
    ```bash
    gemini --model gemini-2.5-flash # For a faster model
    gemini -m "gemini-pro"         # For the standard Gemini Pro model
    ```

2.  **Using the `/model` command (Interactive Mode):**
    If you are in the interactive mode of the Gemini CLI, you can type `/model` and follow the prompts to select a different model.

3.  **Setting an Environment Variable:**
    You can set the `GEMINI_MODEL` environment variable to your preferred model. This will make your choice persistent across sessions without needing the flag every time.
    ```bash
    export GEMINI_MODEL="gemini-2.5-pro"
    ```
    To make this permanent, you can add this line to your shell's configuration file (e.g., `.bashrc`, `.zshrc`).

4.  **Configuring in `settings.json`:**
    The Gemini CLI uses JSON settings files for configuration. You can set the `model` key within these files. Command-line flags generally take precedence over settings in these files.

Please choose the method that best suits your workflow.
